Installation

Introduction

Datagate est composé de nombreux services et une "installation" par conteneurs permet de rendre facilement accessible et reproductible sa mise en place.

Prérequis

Code source

Il vous faudra le code source des conteneurs pour pouvoir les exécuter.

Actuellement ce code est dans une Pull Request (GR) sur le dépôt GitHub de Datagate, voici le lien : PR

Il vous faudra prendre le dernier commit de cette PR et cliquer sur "Browse files" pour accéder à l'arborescence du code source des conteneurs. Dans le menu de droite, cliquez sur "Code" puis "Download ZIP" pour télécharger le code source des conteneurs.

Seul le dossier containers est utilisé pour l'installation.

Installer Datagate chez un hébergeur

L'installation a été testée avec Debian Trixie.

Il est indispensable de le lancer en "rootless" : il faut vous connecter avec un utilisateur qui est sudoer. Certains hébergeurs crée l'utilisateur automatiquement sinon de multiple guides existent sur internet pour créer un utilisateur sudoer.

Attention

Lancer les conteneurs en tant que root entrainera des erreurs car des configurations réseaux complémentaires sont nécessaires

Dans un premier temps, nous allons déposer le code source des conteneurs sur le serveur que vous avez chez un hébergeur.

On utilisera scp pour cela, utilitaire disponible sur la plupart des systèmes d'exploitation. voici un exemple de commande à exécuter depuis votre terminal local, ici le terminal est PowerShell sur Windows : 

scp -r -i .\.ssh\id_pubkey C:\Users\rmugnier\Downloads\datagate\datagate\containers linuxuser@207.148.80.238:~/
De nombreux paramètres sont à adapter selon votre configuration, voici une explication de ceux à remplacer dans l'exemple : - .\.ssh\id_pubkey : chemin vers votre clé publique SSH que vous avez configuré chez votre hébergeur - C:\Users\rmugnier\Downloads\datagate\datagate\containers : chemin vers le dossier containers que vous avez téléchargé depuis la PR - linuxuser : nom de l'utilisateur que vous utilisez pour vous connecter à votre serveur - 207.148.80.238 : adresse IP de votre serveur

Connectez-vous ensuite à votre serveur avec la même clé SSH : 

ssh -i .\.ssh\id_pubkey linuxuser@207.148.80.238

On s'assuera que les paquets présents sur votre serveur sont à jour avec les commandes suivantes : 

sudo apt update
sudo apt upgrade -y

On procède ensuite à l'installation de podman qui est le moteur de conteneurs utilisé pour exécuter les conteneurs de Datagate : 

sudo apt -y install podman

On se positionne ensuite dans le dossier containers que vous avez téléchargé et transféré sur votre serveur : 

cd ~/containers/

Il est fort possible que votre hébergeur a paramétré votre serveur pour que les ports soient bloqués par défaut, on peut utiliser un parefeu simple comme ufw pour ouvrir les ports nécessaires au bon fonctionnement de Datagate : 

sudo apt install ufw
sudo ufw allow 22/tcp
sudo ufw allow 8081:8085/tcp
sudo ufw enable
sudo ufw status

Manipulation de podman

On peut ensuite lancer les conteneurs de Datagate avec la commande suivante :

podman kube play --replace \
--configmap=./datagate-stack-webhookd/configmap.yaml \
--configmap=./datagate-stack-common/apachephp-configmap.yaml \
--configmap=./datagate-stack-common/github-configmap.yaml \
./datagate-stack-kube.yaml

Cette commande permet de nettoyer les conteneurs et ressources réseaux créés par un précédent lancement :

podman kube down ./datagate-stack-kube.yaml \
&& podman volume prune -f \
&& podman network prune -f

Perte de données irréversible

Cette commande supprime toutes les données présentes dans les volumes, comme par exemple la base de données.
